# Contributing to Project-AQUASYNC

First off, thank you for considering contributing to AQUASYNC! It's people like you that make it such a great tool.

## How Can I Contribute?

### Reporting Bugs
- Use a clear and descriptive title.
- Describe the exact steps which reproduce the problem.
- Include details about your environment (OS, ROS 2 version, etc.).

### Suggesting Enhancements
- Explain the behavior you would like to see and why.
- Provide specific examples to demonstrate the step.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Make sure your code follows the existing style.
5. Issue a pull request!

## Code Style
- Follow the official [ROS 2 Developer Guide](https://docs.ros.org/en/humble/The-Docs/Developer-Guide.html).
- For Python: Follow [PEP 8](https://www.python.org/dev/peps/pep-0008/).
- For C++: Follow [Google C++ Style Guide](https://google.github.io/styleguide/cppguide.html).
